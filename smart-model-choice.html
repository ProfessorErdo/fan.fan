The Problem

The Limitation of Rule-Based Recommendations

Traditional product recommendations rely on static rules, collaborative filtering, or simple association mining. These approaches fail to capture temporal patterns in customer behavior, sequential dependencies, and city-specific preferences.

Customers don't buy products in isolation—their purchases form sequences with meaningful patterns. Understanding these sequences requires sophisticated deep learning models that can handle long-range dependencies and contextual information.

  • ✕ Static rules miss temporal purchase patterns
  • ✕ One-size-fits-all models ignore city differences
  • ✕ No understanding of purchase sequence context
  • ✕ Difficulty handling variable-length histories
  • ✕ Flop model identification is reactive, not proactive

System Design

Technical Architecture

End-to-end deep learning pipeline for sequence-based product recommendation

Data Sources

  • Transactions: Member-level purchases
  • Model List: Listed models by store
  • Tree View: Product hierarchy
  • Life Stage: Model lifecycle phase

Sequence Processing

  • Tokenization: Model code → token ID
  • Max Length: 32 time steps
  • Padding: <PAD> token
  • Separation: <SEP> token

Model Architecture

  • Embedding: 256 dimensions
  • Attention: 2 heads, 2 layers
  • Vocab Size: 12,424 tokens
  • Drop Rate: 0.0 (no dropout)

Output & Analysis

  • Recommendations: Top-N models per customer
  • Flop Stats: Underperforming models

  • City Reports: Localized insights
Implementation

Data Pipeline

Six-stage processing pipeline from raw transactions to recommendations

Step 1: Process Listed Models

Extract and process listed model files by city, prepare model inventory data for training and validation

Step 2: Extract Model Lists

Query top models by city from transaction database with quantity and turnover filters, extract for Beijing/Nanjing/Hangzhou

Step 3: Build Sequences

Construct purchase sequences from member transactions, tokenize model codes, create training/validation datasets (1.5M samples)

Step 4: Train Models

Train city-specific transformer models with PyTorch, 90/10 train/val split, save model weights to models directory

Step 5: Generate Recommendations

Load trained models, predict next model probabilities for customers, generate ranked recommendation lists

Step 6: Analysis & Reporting

Query flop models, generate store missing summary, analyze store performance, create city-specific reports
