The Problem

The Complexity of Range Planning

Retail assortment planning is a high-stakes decision affecting revenue, inventory costs, and customer satisfaction. Traditional approaches rely on manual analysis, historical intuition, and one-size-fits-all templates that fail to capture store-specific dynamics.

The challenge is compounded by seasonal variations (different ranges for Spring/Summer vs. Autumn/Winter), space constraints (linear meter limits), and the need to balance iconic products with local preferences.

  • ✕ Manual range selection lacks quantitative rigor
  • ✕ Different stores have different customer bases
  • ✕ Seasonal variations require separate planning
  • ✕ Space constraints (linear meters) must be respected
  • ✕ Compulsory products must be included regardless of performance

The Solution

A Data-Driven Approach

End-to-end assortment planning pipeline from comparable store identification to final recommendations

1

Query Store Performance

Extract family-level turnover data by store and season from data warehouse, calculate turnover per linear meter per day

2

Calculate Pareto & Range

For each store-season combination, calculate cumulative turnover percentage and identify families contributing to 80% of turnover

3

Identify Comparable Stores

Find 3-4 most similar stores using cosine distance on external features (geospatial + POI + demographics + traffic)

4

Generate Recommendations

Combine prioritized families from comparable stores, rank by turnover per linear meter, generate initial recommendations

5

Apply Linear Meter Constraint

Constrain recommendations to fit store's total linear meter capacity (typically 60-70% of total LM to allow flexibility)

6

Apply Compulsory Matching

Enforce business rules: add mandatory product families (iconic products, strategic items) regardless of performance metrics

System Design

Technical Architecture

Modular pipeline with six core components working in sequence

Data Sources

  • Turnover Data: Family-level by store
  • Range Choice: Seasonal product ranges
  • Store Data: Linear meters, status
  • Comparable Stores: Pre-calculated similarity

Pareto Analysis

  • Cumulative %: Running turnover total
  • Threshold: 80% cutoff (configurable)
  • Priority: High-turnover families
  • Seasonal: Separate per season

Recommendation Engine

  • Mode: Single or Multiple stores
  • Distance: Cosine similarity
  • Features: External all geo
  • Threshold: 0.8 cumulative

Post-Processing

  • LM Constraint: 60-70% of total
  • Compulsory: Mandatory families
  • Validation: Outlier analysis
  • Reports: Tableau-ready exports
Implementation

Data Pipeline

Six-stage processing pipeline from raw data to final recommendations

Step 1: Query Store Performance

Extract family turnover by store and season from warehouse, calculate turnover per linear meter per day metric

Step 2: Combine Ranges & Performance

Join range choice data with performance metrics, calculate cumulative turnover percentage for Pareto analysis

Step 3: Recommend Ranges

Load comparable stores (3-4 per target), extract prioritized families from all comparable stores, rank by performance

Step 4: Apply LM Constraint

Calculate recommended linear meters vs. store total, apply constraint factor (0.6-0.7), remove lowest performers

Step 5: Apply Compulsory Matching

Load compulsory matching list by season, add mandatory families, validate against business rules, remove invalid entries

Step 6: Generate Reports

Export to CSV for Tableau dashboards, generate family tree view, perform outlier analysis, validate recommendations
