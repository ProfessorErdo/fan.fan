为什么重要

将产品生命周期管理从直觉决策转向数据驱动战略

战略规划

准确预测产品何时经历生命周期阶段转换（导入→成长→成熟→衰退），实现主动库存管理和资源分配。

成本优化

通过预测产品生命周期终点减少浪费和过量库存，最小化清仓损失，基于数据驱动的寿命估算优化采购周期。

组合管理

按周转模式和季节行为细分产品，早期识别高表现者，对产品线延伸或淘汰做出明智决策。

收入保护

通过理解每个产品从上市到生命周期终点的轨迹，防止有潜力产品缺货，避免对衰退线过度投资。

问题背景

传统PLC框架的局限性

产品生命周期（PLC）框架是战略营销和产品管理的基石。然而，其实用性受到一个关键缺陷的严重限制：无法准确预测产品何时在各阶段之间转换。

传统方法依赖人工分析、历史类比或简单启发式——这些方法无法捕捉现代零售数据中的复杂模式。

  • ✕ 主观评估导致不一致预测
  • ✕ 缺乏生命周期阶段转换的量化指标
  • ✕ 无法高效处理大规模周转数据
  • ✕ 无系统方法识别季节性与结构性模式
  • ✕ 被动而非主动决策

解决方案

数据驱动方法

将原始周转数据转化为可执行寿命预测的端到端机器学习流程

1

DTW模式识别

Dynamic Time Warping测量产品周转曲线之间的相似性，识别具有相似生命周期模式的产品，不受时间变化影响

2

季节智能

使用方差分解量化季节强度，区分季节性产品（滑雪装备）和全年表现者（基础服装）

3

产品细分

K-means聚类按周转行为分组产品，使模型学习不同产品类别的独特生命周期模式

4

预测建模

Gradient boosting模型（XGBoost、LightGBM、CatBoost）在工程特征上训练，通过Optuna进行超参数优化预测寿命

5

交互仪表板

基于Streamlit的可视化工具使业务用户探索预测、查看周转趋势、理解模型推理

实现细节

数据流程

从原始数据到预测的八阶段处理流程

步骤0：数据提取

从企业数据仓库（ODPS）加载周转数据和产品层级

步骤1：DTW距离计算

计算Dynamic Time Warping距离到平均和标准周转曲线用于模式识别

步骤2：季节强度

分解时间序列，计算强度 = var(seasonal) / var(raw)，分类为弱/中/强

步骤3：K-means聚类

使用DTW距离和强度指标细分产品，通过elbow分析确定最优k

步骤4：特征组装

合并所有特征，从周转曲线计算寿命，按聚类和类型分层划分

步骤5：模型训练

训练gradient boosting模型，使用Optuna超参数优化和早停

步骤6：预测生成

应用最佳模型为活跃产品生成寿命预测

步骤7：仪表板导出

准备数据用于交互Streamlit可视化，包含趋势图和特征分解

特征工程

关键预测因子

工程特征以捕捉产品生命周期模式和季节行为

模式特征

  • →
    mean_distance DTW距离到平均周转曲线
  • →
    std_distance DTW距离到标准差曲线
  • →
    cluster_label K-means聚类分配（0到k-1）

季节特征

  • →
    seasonal_intensity var(seasonal) / var(raw)比率
  • →
    seasonal_class 弱/中/强分类
  • →
    peak_week 最大季节效应周

周转特征

  • →
    total_turnover 产品生命周期累计销售
  • →
    peak_turnover 最大周周转
  • →
    turnover_variance 销售波动度量
  • →
    weeks_to_peak 从上市到峰值销售时间

产品属性

  • →
    sport_family 产品类别（跑步、徒步等）
  • →
    typology 产品类型分类
  • →
    price_range 价格层级分类
