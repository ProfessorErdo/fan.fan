The Problem

The Complexity of Site Selection

Retail site selection has traditionally relied on expert experience and limited data. Predicting the economic performance of a new store location requires understanding complex interactions between geographic, demographic, and competitive factors that vary dramatically across different areas and distances.

The challenge is compounded by multiple factors: varying catchment area definitions (3km, 5km, 10-minute drive, living area), diverse data sources (population grids, traffic counts, POI data), and the need to predict multiple performance metrics simultaneously.

  • ✕ Experience-based decisions lack quantitative rigor
  • ✕ Difficulty aggregating multi-source geospatial data
  • ✕ No systematic way to compare location potential
  • ✕ Inability to predict performance across different area investments
  • ✕ Missing visualization tools for geographic business intelligence

End-to-end geospatial ML pipeline that transforms location data into actionable site selection insights

System Design

Technical Architecture

Modular pipeline with three core functions: weight calculation, single-output prediction, and multi-target prediction

Data Sources

  • POI Data: Competitors, malls, sports
  • Population: Demographics, income, age
  • Traffic: Hourly foot traffic patterns
  • Store Data: Internal performance metrics

Distance Aggregation

  • 250M: Grid-based (250m × 250m)
  • 3KM/5KM: Circular catchment
  • 10MIN: Drive-time polygon
  • LA: Living area boundaries
  • Combined: Hybrid aggregations

Model Training

  • Weight Calc: Lasso regression
  • Single Output: XGB/CatBoost (TO or TO/SQM)
  • Multi-Target: Simultaneous prediction
  • Optimization: Optuna TPE (200 trials)

Output & Visualization

  • Driver Weights: CSV for GIS integration
  • Predictions: TO and TO/SQM by area
  • Intervals: Confidence ranges
  • Dashboards: Tableau visualization
Implementation

Data Pipeline

Six-stage processing pipeline from raw geospatial data to site selection recommendations

Step 0: Data Collection

Gather external data (POI, population, traffic) and internal store performance data for top 30 Chinese cities

Step 1: Preprocessing

Clean and standardize data: impute missing values, standardize city names and building types, calculate store age and sales area ratio, create city tiers

Step 2: Distance Aggregation

Aggregate geospatial data into 9 distance definitions: 250M, 3KM, 5KM, 10MIN, LA, and 4 combined variants (poi+population at distance + traffic in 250m grids)

Step 3: Feature Selection

Train base models, calculate permutation importance, rank features by importance, remove low-importance features and random noise features

Step 4: Model Training

Optuna hyperparameter optimization (200 trials, TPE sampler), 5-fold stratified CV, early stopping (100 rounds), save best models for each distance × model combination

Step 5: Prediction & Reporting

Generate predictions for existing stores (TO, TO/SQM, intervals), predict new store performance, create consolidated reports, export to Tableau dashboards
