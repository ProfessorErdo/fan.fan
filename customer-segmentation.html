The Problem

The Limitation of Traditional Segmentation

Traditional customer segmentation relies on demographics (age, gender, location) or simple RFM (Recency, Frequency, Monetary) analysis. These approaches fail to capture the rich behavioral patterns in modern transaction data.

Customers who look similar demographically may have completely different purchase behaviors. Conversely, customers from different demographics may share similar shopping patterns. Understanding true customer segments requires analyzing actual purchase behavior at the transaction level.

  • ✕ Demographics don't predict purchase behavior
  • ✕ Manual segmentation is subjective and inconsistent
  • ✕ Difficulty handling high-dimensional behavioral data
  • ✕ No systematic way to determine optimal segment count
  • ✕ Segments become outdated without regular refresh

The Solution

A Machine Learning Approach

K-Means clustering with PCA dimensionality reduction for customer segmentation

1

Query Transaction Data

Extract member transactions from data warehouse with date range and city filters, filter positive item amounts

2

Feature Engineering

Aggregate transactions to member level: total spend, frequency, unique categories, average basket value, recency

3

Data Preprocessing

Handle missing values, remove outliers, standardize features using StandardScaler for K-Means compatibility

4

PCA Dimensionality Reduction

Apply Principal Component Analysis to reduce dimensionality, visualize data in 2D/3D, retain 95% variance

5

K-Means Clustering

Find optimal K using elbow method and silhouette scores, run K-Means with 1000 iterations for convergence

6

Segment Profiling & Visualization

Profile each cluster with descriptive statistics, create visualizations with Missingno, Seaborn, and Matplotlib

System Design

Technical Architecture

End-to-end customer segmentation pipeline from raw transactions to actionable segments

Data Sources

  • Transactions: Member-level purchases
  • Time Range: Configurable start/end dates
  • Geography: City-level filtering
  • Grain: Purchase sports ID or family ID

Preprocessing

  • Aggregation: Transaction → Member level
  • Scaling: StandardScaler normalization
  • Outliers: IQR-based removal
  • Missing: Imputation or removal

Clustering Engine

  • Algorithm: K-Means
  • Iterations: 1,000 (guaranteed convergence)
  • Optimal K: Elbow + Silhouette analysis
  • Validation: Silhouette score

Output & Analysis

  • Segment Profiles: Descriptive statistics per cluster
  • Visualizations: PCA plots, cluster scatter
  • Missing Data: Missingno matrix
  • Reports: City-specific insights
